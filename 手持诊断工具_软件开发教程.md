## 1.1 系统架构回顾


## 1.2 软件技术栈总览

| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 子机 OS | Zephyr RTOS | 需求文档指定 |
| 子机 MCU | ESP32-S3 | 微雪开发板，自带 TWAI CAN 控制器 |
| 子机协议栈 | CANopenNode | 开源 CANopen 实现 |
| 子机通信 | Wi-Fi AP + UDP | 热点模式，高频心跳 |
| 母机 OS | Linux (Debian/Ubuntu) | Powkiddy 掌机或 Linux 开发板 |
| 母机 UI | Python 3 + PyQt5 | 快速原型，支持触摸屏 |
| 母机通信 | Wi-Fi STA + UDP/TCP | 连接子机热点 |
| 通信协议 | JSON over UDP | 轻量、易调试 |

## 1.3 自定义通信协议（母机 ↔ 子机）

在开始写代码之前，需要先定义母机和子机之间的通信协议。我们采用 JSON 格式通过 UDP 传输。

### 消息格式

```json
{
  "cmd": "命令类型",
  "seq": 1234,
  "ts": 1680000000,
  "payload": { ... }
}
```

### 命令列表

| 命令 | 方向 | 用途 | payload 示例 |
|------|------|------|-------------|
| `heartbeat` | 母机→子机 | 心跳包 | `{}` |
| `sdo_read` | 母机→子机 | 读取对象字典 | `{"node":1,"index":0x6041,"sub":0}` |
| `sdo_write` | 母机→子机 | 写入对象字典 | `{"node":1,"index":0x6060,"sub":0,"data":3}` |
| `pdo_tx` | 母机→子机 | 发送 PDO 控制字 | `{"node":1,"data":[0x0F,0x00]}` |
| `pdo_rx` | 子机→母机 | 接收 PDO 状态数据 | `{"node":1,"data":[0x37,0x00,0x00,0x00]}` |
| `motor_status` | 子机→母机 | 电机状态上报 | `{"node":1,"current":2.5,"voltage":24,"speed":1500,"position":360,"fault":0}` |
| `jog_start` | 母机→子机 | 点动启动 | `{"node":1,"direction":"cw","speed":500}` |
| `jog_stop` | 母机→子机 | 点动停止 | `{"node":1}` |
| `enable` | 母机→子机 | 电机使能 | `{"node":1}` |
| `disable` | 母机→子机 | 电机去使能 | `{"node":1}` |
| `estop` | 母机→子机 | 急停 | `{}` |
| `ota_start` | 母机→子机 | 开始 OTA | `{"size":65536,"md5":"abc123..."}` |
| `ota_chunk` | 母机→子机 | OTA 数据分片 | `{"offset":0,"data":"base64..."}` |
| `ota_verify` | 母机→子机 | 请求校验 | `{}` |
| `ota_flash` | 母机→子机 | 请求刷写电机 | `{"node":1}` |
| `ota_status` | 子机→母机 | OTA 进度上报 | `{"state":"idle/receiving/verifying/flashing/done/error","progress":50}` |
| `ack` | 双向 | 确认收到 | `{"ref_seq":1234,"status":"ok/error","msg":"..."}` |

## 2.1 开发环境搭建（macOS）

### 2.1.1 安装系统依赖

```bash

# 安装编译工具链
brew install cmake ninja gperf python3 ccache dtc wget
```

> **注意：** macOS 上不需要 `gcc-multilib`、`g++-multilib`、`libsdl2-dev` 等 Linux 专属包。ESP32 的交叉编译工具链需要单独从 Espressif 下载。

### 2.1.2 获取 Zephyr 源码

```bash
# 创建项目目录
mkdir -p ~/esp32-can-dongle && cd ~/esp32-can-dongle

# 初始化 west 工程（Zephyr 官方推荐的构建工具）
west init -m https://github.com/zephyrproject-rtos/zephyr --mr v3.6.0
west update

# 安装 Zephyr Python 依赖
# 如果系统 Python 报 "externally-managed-environment" 错误，先创建虚拟环境：
python3 -m venv ~/zephyr-env
source ~/zephyr-env/bin/activate
pip install -r zephyr/scripts/requirements.txt
```

### 2.1.3 安装 ESP32 交叉编译工具链

> **重要：** Zephyr 3.6.0 不支持 `west espressif install`，需要手动下载 Espressif 的芯片专用工具链。

```bash
# 创建工具链目录
mkdir -p ~/.espressif/tools/zephyr

# 根据你的 Mac 架构下载对应的工具包：
# Apple Silicon (M1/M2/M3/M4)：

# Intel Mac：
# wget https://github.com/espressif/crosstool-NG/releases/download/esp-12.2.0_20230208/xtensa-esp32s3-elf-12.2.0_20230208-x86_64-apple-darwin.tar.xz

# 解压到工具链目录
tar xf xtensa-esp32s3-elf-12.2.0_20230208-aarch64-apple-darwin.tar.xz -C ~/.espressif/tools/zephyr/

# 验证编译器
ls ~/.espressif/tools/zephyr/xtensa-esp32s3-elf/bin/xtensa-esp32s3-elf-gcc
```

### 2.1.4 配置环境变量

在 `~/.zshrc` 中添加（macOS 默认 shell 是 zsh）：

```bash
echo 'export PATH="$HOME/.espressif/tools/zephyr/xtensa-esp32s3-elf/bin:$PATH"' >> ~/.zshrc
```

> **注意：** 必须下载芯片专用版工具链（`xtensa-esp32s3-elf`），不要用 Espressif 的统一版（`xtensa-esp-elf`），后者在 Zephyr 3.6.0 上会因头文件路径不兼容导致编译失败。

## 2.3 Wi-Fi AP 配对模块

### 2.3.1 wifi_ap.h

```c
#ifndef WIFI_AP_H
#define WIFI_AP_H

#include <zephyr/kernel.h>

// Wi-Fi AP 配置
```

### 2.3.2 wifi_ap.c

```c
#include "wifi_ap.h"
#include <zephyr/logging/log.h>
#include <zephyr/net/wifi_mgmt.h>
#include <zephyr/net/net_if.h>

LOG_MODULE_REGISTER(wifi_ap, CONFIG_LOG_DEFAULT_LEVEL);

static wifi_ap_state_t ap_state = WIFI_AP_IDLE;
static struct net_if *wifi_iface;

// 回调：Wi-Fi AP 状态变化
static void wifi_ap_status_cb(struct net_if *iface,
                               struct net_mgmt_event_callback *cb,
                               uint32_t mgmt_event,
                               struct net_event_info *info)
{
    switch (mgmt_event) {
    case NET_EVENT_WIFI_AP_ENABLE_RESULT:
        if (info && info->status == 0) {
            LOG_INF("Wi-Fi AP 启动成功，SSID: %s", WIFI_AP_SSID);
            ap_state = WIFI_AP_RUNNING;
        } else {
            LOG_ERR("Wi-Fi AP 启动失败");
            ap_state = WIFI_AP_ERROR;
        }
        break;

    case NET_EVENT_WIFI_AP_STA_CONNECTED:
        LOG_INF("母机已连接!");
        ap_state = WIFI_AP_CLIENT_CONNECTED;
        break;

    case NET_EVENT_WIFI_AP_STA_DISCONNECTED:
        LOG_WRN("母机断开连接");
        ap_state = WIFI_AP_RUNNING;
        break;

    default:
        break;
    }
}

int wifi_ap_init(void)
{
    static struct net_mgmt_event_callback wifi_ap_cb;

    // 获取默认 Wi-Fi 网络接口
    wifi_iface = net_if_get_default();
    if (!wifi_iface) {
        LOG_ERR("未找到网络接口");
        return -ENODEV;
    }

    // 注册 Wi-Fi 事件回调
    net_mgmt_init_event_callback(&wifi_ap_cb, wifi_ap_status_cb,
                                  NET_EVENT_WIFI_AP_ENABLE_RESULT |
                                  NET_EVENT_WIFI_AP_STA_CONNECTED |
                                  NET_EVENT_WIFI_AP_STA_DISCONNECTED);
    net_mgmt_add_event_callback(&wifi_ap_cb);

    // 配置 AP 参数
    struct wifi_connect_req_params ap_params = {
        .ssid = WIFI_AP_SSID,
        .ssid_length = strlen(WIFI_AP_SSID),
        .psk = WIFI_AP_PASSWORD,
        .psk_length = strlen(WIFI_AP_PASSWORD),
        .channel = WIFI_AP_CHANNEL,
        .security = WIFI_SECURITY_TYPE_PSK,
        .bandwidth = WIFI_FREQ_BAND_2_4_GHZ,
    };

    ap_state = WIFI_AP_STARTING;

    // 设置为 AP 模式
    int ret = net_if_set_wifi_ap(wifi_iface, &ap_params);
    if (ret) {
        LOG_ERR("启动 AP 模式失败: %d", ret);
        ap_state = WIFI_AP_ERROR;
        return ret;
    }

    LOG_INF("Wi-Fi AP 正在启动...");
    return 0;
}

wifi_ap_state_t wifi_ap_get_state(void)
{
    return ap_state;
}

bool wifi_ap_is_client_connected(void)
{
    return (ap_state == WIFI_AP_CLIENT_CONNECTED);
}
```

**关键点解释：**
- AP 模式意味着 Dongle 自建热点，母机主动连上来，避免了依赖外部路由器
- SSID 可以包含设备编号（如 `CAN_Dongle_01`），方便在多台 Dongle 存在时区分
- `MAX_CONN=1` 确保只有一个客户端连接，防止多设备串扰

### 2.4.2 udp_comm.c

```c
#include "udp_comm.h"
#include <zephyr/logging/log.h>
#include <zephyr/net/socket.h>
#include <string.h>

LOG_MODULE_REGISTER(udp_comm, CONFIG_LOG_DEFAULT_LEVEL);

static int udp_sock = -1;
static struct sockaddr_in remote_addr;
static udp_recv_cb_t recv_callback;
static char recv_buf[UDP_BUF_SIZE];
static K_THREAD_STACK_DEFINE(recv_thread_stack, 2048);
static struct k_thread recv_thread_data;

// 接收线程：持续监听 UDP 数据
static void udp_recv_thread(void *p1, void *p2, void *p3)
{
    ARG_UNUSED(p1); ARG_UNUSED(p2); ARG_UNUSED(p3);

    struct sockaddr_in src_addr;
    socklen_t addrlen = sizeof(src_addr);

    LOG_INF("UDP 接收线程已启动，监听端口 %d", UDP_LOCAL_PORT);

    while (1) {
        ssize_t len = recvfrom(udp_sock, recv_buf, sizeof(recv_buf) - 1, 0,
                               (struct sockaddr *)&src_addr, &addrlen);
        if (len > 0) {
            recv_buf[len] = '\0';  // 确保字符串终止
            char src_ip[16];
            inet_ntoa(src_addr.sin_addr, src_ip, sizeof(src_ip));

            LOG_DBG("收到 %d 字节，来源 %s:%d", len, src_ip,
                    ntohs(src_addr.sin_port));

            if (recv_callback) {
                recv_callback(recv_buf, len, src_ip,
                              ntohs(src_addr.sin_port));
            }
        }
    }
}

int udp_comm_init(udp_recv_cb_t recv_cb)
{
    recv_callback = recv_cb;

    // 创建 UDP socket
    udp_sock = socket(AF_INET, SOCK_DGRAM, IPPROTO_UDP);
    if (udp_sock < 0) {
        LOG_ERR("创建 socket 失败: %d", errno);
        return -errno;
    }

    // 绑定本地地址
    struct sockaddr_in local_addr = {
        .sin_family = AF_INET,
        .sin_port = htons(UDP_LOCAL_PORT),
        .sin_addr.s_addr = htonl(INADDR_ANY),
    };

    if (bind(udp_sock, (struct sockaddr *)&local_addr, sizeof(local_addr))) {
        LOG_ERR("绑定端口失败: %d", errno);
        close(udp_sock);
        return -errno;
    }

    // 配置远程地址（母机）
    remote_addr.sin_family = AF_INET;
    remote_addr.sin_port = htons(UDP_REMOTE_PORT);
    inet_pton(AF_INET, UDP_REMOTE_IP, &remote_addr.sin_addr);

    // 启动接收线程（高优先级，确保心跳不丢）
    k_thread_create(&recv_thread_data, recv_thread_stack,
                    K_THREAD_STACK_SIZEOF(recv_thread_stack),
                    udp_recv_thread, NULL, NULL, NULL,
                    K_PRIO_PREEMPT(5), 0, K_NO_WAIT);
    k_thread_name_set(&recv_thread_data, "udp_recv");

    LOG_INF("UDP 通信初始化完成");
    return 0;
}

int udp_comm_send(const char *data, size_t len)
{
    if (udp_sock < 0) return -ENOTCONN;

    ssize_t sent = sendto(udp_sock, data, len, 0,
                          (struct sockaddr *)&remote_addr,
                          sizeof(remote_addr));
    if (sent < 0) {
        LOG_ERR("发送失败: %d", errno);
        return -errno;
    }
    return 0;
}

int udp_comm_send_json(const char *json_str)
{
    return udp_comm_send(json_str, strlen(json_str));
}
```

**关键点解释：**
- 接收线程运行在优先级 5（数值越小优先级越高），确保心跳包能被及时处理
- 使用 `sendto`/`recvfrom` 无连接模式，减少延迟
- 每次收到数据都触发回调，上层逻辑统一处理

### 2.5.4 canopen_stack.c（核心实现）

```c
#include "canopen_stack.h"
#include "CO_OD.h"
#include <zephyr/logging/log.h>
#include <zephyr/drivers/can.h>
#include <zephyr/sys/byteorder.h>
#include <string.h>

LOG_MODULE_REGISTER(canopen, CONFIG_LOG_DEFAULT_LEVEL);

// 获取 CAN 设备
static const struct device *can_dev;

// SDO 传输状态
static struct {
    struct k_sem sem;           // 同步信号量
    uint8_t response[8];        // 响应数据
    size_t response_len;
    int status;                 // 0=成功, 负数=错误
} sdo_transfer;

// PDO 接收回调
static void (*pdo_recv_callback)(uint16_t cob_id, const uint8_t *data, size_t len);

// CAN 接收回调
static void can_rx_callback(const struct device *dev, struct can_frame *frame,
                            void *user_data)
{
    uint8_t function_code = (frame->id >> 7) & 0x0F;
    uint8_t node_id = frame->id & 0x7F;

    LOG_DBG("CAN RX: ID=0x%03X, DLC=%d, data=[%02X %02X %02X %02X]",
            frame->id, frame->dlc,
            frame->data[0], frame->data[1],
            frame->data[2], frame->data[3]);

    // 分发到不同处理器
    switch (function_code) {
    case 0x0B:  // SDO Server → Client（SDO 响应）
        if (sdo_transfer.sem) {
            memcpy(sdo_transfer.response, frame->data, frame->dlc);
            sdo_transfer.response_len = frame->dlc;
            sdo_transfer.status = 0;
            k_sem_give(&sdo_transfer.sem);
        }
        break;

    case 0x03:  // TPDO1（节点 → 主站）
    case 0x05:  // TPDO2
    case 0x07:  // TPDO3
        if (pdo_recv_callback) {
            pdo_recv_callback(frame->id, frame->data, frame->dlc);
        }
        break;

    case 0x0E:  // Heartbeat / NMT 错误控制
        LOG_DBG("节点 %d 心跳: %02X", node_id, frame->data[0]);
        break;

    case 0x01:  // EMCY 紧急报文
        LOG_WRN("节点 %d 紧急报文: %02X%02X", node_id,
                frame->data[0], frame->data[1]);
        break;

    default:
        break;
    }
}

int canopen_init(uint8_t node_id)
{
    // 获取 CAN 设备（在设备树中定义为 &twai）
    can_dev = DEVICE_DT_GET(DT_NODELABEL(twai));
    if (!device_is_ready(can_dev)) {
        LOG_ERR("CAN 设备未就绪");
        return -ENODEV;
    }

    // 配置 CAN 波特率：1000 kbps（机器人常用）
    struct can_timing timing;
    int ret = can_calc_timing(can_dev, &timing, 1000000, 875);
    if (ret) {
        LOG_ERR("CAN 波特率计算失败: %d", ret);
        return ret;
    }

    ret = can_set_timing(can_dev, &timing);
    if (ret) {
        LOG_ERR("设置 CAN 时序失败: %d", ret);
        return ret;
    }

    // 设置过滤器：接收所有帧（开发阶段）
    struct can_filter filter = {
        .id = 0,
        .mask = 0,
        .flags = CAN_FILTER_DATA,
    };
    can_add_rx_filter(can_dev, can_rx_callback, NULL, &filter);

    // 启动 CAN 控制器
    ret = can_start(can_dev);
    if (ret) {
        LOG_ERR("CAN 启动失败: %d", ret);
        return ret;
    }

    // 初始化 SDO 信号量
    k_sem_init(&sdo_transfer.sem, 0, 1);

    LOG_INF("CANopen 栈初始化完成，节点 ID=%d", node_id);
    return 0;
}

// SDO 读取对象字典
int canopen_sdo_read(uint8_t target_node, uint16_t index,
                     uint8_t sub_index, uint8_t *data_out,
                     size_t *data_len, int timeout_ms)
{
    struct can_frame frame = {
        .id = 0x600 + target_node,   // SDO Client → Server COB-ID
        .dlc = 8,
        .flags = 0,
    };

    // SDO 上传请求（Client Command Specifier = 2）
    frame.data[0] = 0x40;           // 上传请求
    frame.data[1] = index & 0xFF;
    frame.data[2] = (index >> 8) & 0xFF;
    frame.data[3] = sub_index;
    frame.data[4] = 0;
    frame.data[5] = 0;
    frame.data[6] = 0;
    frame.data[7] = 0;

    sdo_transfer.status = -EAGAIN;
    k_sem_reset(&sdo_transfer.sem);

    int ret = can_send(can_dev, &frame, K_MSEC(10), NULL, NULL);
    if (ret) {
        LOG_ERR("SDO 读发送失败: %d", ret);
        return ret;
    }

    // 等待响应
    ret = k_sem_take(&sdo_transfer.sem, K_MSEC(timeout_ms));
    if (ret) {
        LOG_ERR("SDO 读超时 (节点 %d, 索引 0x%04X)", target_node, index);
        return -ETIMEDOUT;
    }

    // 解析响应
    uint8_t ccs = (sdo_transfer.response[0] >> 5) & 0x03;
    if (ccs == 3) {  // Abort
        LOG_ERR("SDO 读被中止");
        return -EIO;
    }

    // 判断数据长度（快速传输或段传输）
    bool exp = (sdo_transfer.response[0] & 0x02) != 0;  // Expedited
    if (exp) {
        uint8_t n = (sdo_transfer.response[0] >> 2) & 0x03;
        *data_len = 4 - n;
        memcpy(data_out, &sdo_transfer.response[4], *data_len);
    } else {
        // 标准传输：长度在 bytes 4-7
        *data_len = sys_get_le32(&sdo_transfer.response[4]);
        if (*data_len > 4) {
            // 需要多段传输（此处简化，实际应循环）
            LOG_WRN("SDO 非快速传输，数据较长");
        }
        memcpy(data_out, &sdo_transfer.response[4], *data_len);
    }

    return 0;
}

// SDO 写入对象字典
int canopen_sdo_write(uint8_t target_node, uint16_t index,
                      uint8_t sub_index, const uint8_t *data,
                      size_t data_len, int timeout_ms)
{
    struct can_frame frame = {
        .id = 0x600 + target_node,
        .dlc = 8,
        .flags = 0,
    };

    if (data_len <= 4) {
        // Expedited Download（快速下载，最多4字节）
        frame.data[0] = 0x23 | ((4 - data_len) << 2);  // CCS=1, n
        frame.data[1] = index & 0xFF;
        frame.data[2] = (index >> 8) & 0xFF;
        frame.data[3] = sub_index;
        memset(&frame.data[4], 0, 4);
        memcpy(&frame.data[4], data, data_len);
    } else {
        // 长数据需要分段传输（此处用 Block Download 简化表示）
        LOG_WRN("SDO 长数据写入需要分段，长度=%d", data_len);
        return -ENOTSUP;
    }

    sdo_transfer.status = -EAGAIN;
    k_sem_reset(&sdo_transfer.sem);

    int ret = can_send(can_dev, &frame, K_MSEC(10), NULL, NULL);
    if (ret) {
        LOG_ERR("SDO 写发送失败: %d", ret);
        return ret;
    }

    ret = k_sem_take(&sdo_transfer.sem, K_MSEC(timeout_ms));
    if (ret) {
        LOG_ERR("SDO 写超时");
        return -ETIMEDOUT;
    }

    // 检查是否收到确认
    if (sdo_transfer.response[0] != 0x60) {  // 0x60 = 下载成功确认
        LOG_ERR("SDO 写确认异常: %02X", sdo_transfer.response[0]);
        return -EIO;
    }

    return 0;
}

// 发送 PDO
int canopen_pdo_send(uint16_t cob_id, const uint8_t *data, size_t len)
{
    struct can_frame frame = {
        .id = cob_id,
        .dlc = MIN(len, 8),
        .flags = 0,
    };
    memcpy(frame.data, data, frame.dlc);

    return can_send(can_dev, &frame, K_MSEC(5), NULL, NULL);
}

// NMT 命令
int canopen_nmt_send(canopen_nmt_cmd_t cmd, uint8_t target_node)
{
    struct can_frame frame = {
        .id = 0x000,  // NMT COB-ID 永远是 0
        .dlc = 2,
        .flags = 0,
    };
    frame.data[0] = (uint8_t)cmd;
    frame.data[1] = target_node;  // 0 = 广播所有节点

    LOG_INF("NMT 命令: cmd=0x%02X, node=%d", cmd, target_node);
    return can_send(can_dev, &frame, K_MSEC(10), NULL, NULL);
}
```

## 2.6 心跳看门狗模块（核心安全机制）

这是整个系统最关键的安全模块。需求文档要求：**母机每 100-200ms 发心跳，子机 500ms 无心跳则急停。**

### 2.6.2 watchdog.c

```c
#include "watchdog.h"
#include "canopen_stack.h"
#include <zephyr/logging/log.h>

LOG_MODULE_REGISTER(watchdog, CONFIG_LOG_DEFAULT_LEVEL);

static uint32_t timeout_ms = 500;
static int64_t last_heartbeat_tick = 0;
static bool is_alive = true;
static bool emergency_triggered = false;

// 看门狗检查线程
static K_THREAD_STACK_DEFINE(wdog_stack, 1024);
static struct k_thread wdog_thread_data;

// 检查线程：最高优先级运行
static void watchdog_thread(void *p1, void *p2, void *p3)
{
    ARG_UNUSED(p1); ARG_UNUSED(p2); ARG_UNUSED(p3);

    LOG_INF("看门狗线程启动，超时阈值=%dms", timeout_ms);

    while (1) {
        // 每 50ms 检查一次（比超时时间短得多，确保精度）
        k_msleep(50);

        if (emergency_triggered) {
            // 已触发急停，保持状态直到收到新心跳
            continue;
        }

        int64_t elapsed = k_uptime_get() - last_heartbeat_tick;

        if (elapsed > timeout_ms) {
            // 心跳超时！执行紧急停机
            LOG_ERR("==========================================");
            LOG_ERR("心跳超时! 已 %lld ms 未收到母机心跳", elapsed);
            LOG_ERR("触发紧急停机!");
            LOG_ERR("==========================================");

            is_alive = false;
            emergency_triggered = true;

            // 执行 NMT Stop + EMCY
            canopen_emergency_stop();
        }
    }
}

int watchdog_init(uint32_t timeout)
{
    timeout_ms = timeout;
    last_heartbeat_tick = k_uptime_get();  // 初始化时间戳
    is_alive = true;
    emergency_triggered = false;

    // 以最高优先级（优先级 1）创建看门狗线程
    k_thread_create(&wdog_thread_data, wdog_stack,
                    K_THREAD_STACK_SIZEOF(wdog_stack),
                    watchdog_thread, NULL, NULL, NULL,
                    K_PRIO_PREEMPT(1),  // 仅次于优先级 0（协作式）
                    0, K_NO_WAIT);
    k_thread_name_set(&wdog_thread_data, "watchdog");

    LOG_INF("心跳看门狗已初始化");
    return 0;
}

void watchdog_feed(void)
{
    last_heartbeat_tick = k_uptime_get();

    if (emergency_triggered) {
        // 收到新心跳，解除急停状态
        LOG_INF("收到母机心跳，解除急停状态");
        emergency_triggered = false;
        is_alive = true;
        // 注意：不会自动重新使能电机，需要母机显式发送 enable 指令
    }
}

bool watchdog_is_alive(void)
{
    return is_alive;
}

uint32_t watchdog_time_since_last_heartbeat(void)
{
    return (uint32_t)(k_uptime_get() - last_heartbeat_tick);
}
```

**关键设计点：**
1. 检查频率 50ms，远低于 500ms 超时阈值，确保精度
2. 线程优先级设为 1（几乎最高），确保不被其他任务阻塞
3. 超时后自动广播 NMT Stop，切断所有节点动力
4. 解除急停后不会自动使能电机，必须母机显式操作——这是安全设计
5. `emergency_triggered` 标志防止反复执行急停

### 2.7.2 command_parser.c

```c
#include "command_parser.h"
#include "canopen_stack.h"
#include "watchdog.h"
#include "ota_manager.h"
#include "udp_comm.h"
#include <zephyr/logging/log.h>
#include <cJSON.h>
#include <string.h>
#include <stdlib.h>

LOG_MODULE_REGISTER(cmd_parser, CONFIG_LOG_DEFAULT_LEVEL);

// 构建 ACK 回复
static int build_ack(char *buf, size_t size, int seq,
                     const char *status, const char *msg)
{
    cJSON *reply = cJSON_CreateObject();
    cJSON_AddStringToObject(reply, "cmd", "ack");
    cJSON_AddNumberToObject(reply, "ref_seq", seq);
    cJSON_AddStringToObject(reply, "status", status);
    if (msg) cJSON_AddStringToObject(reply, "msg", msg);

    char *str = cJSON_PrintUnformatted(reply);
    strncpy(buf, str, size - 1);
    buf[size - 1] = '\0';
    free(str);
    cJSON_Delete(reply);
    return 0;
}

// 处理 SDO 读取
static int handle_sdo_read(cJSON *payload, char *reply, size_t reply_size)
{
    uint8_t node = cJSON_GetObjectItem(payload, "node")->valueint;
    uint16_t index = cJSON_GetObjectItem(payload, "index")->valueint;
    uint8_t sub = cJSON_GetObjectItem(payload, "sub")->valueint;

    uint8_t data[4] = {0};
    size_t data_len = 0;

    int ret = canopen_sdo_read(node, index, sub, data, &data_len, 1000);
    if (ret) {
        return -1;
    }

    // 构建回复
    cJSON *resp = cJSON_CreateObject();
    cJSON_AddStringToObject(resp, "cmd", "sdo_read_result");
    cJSON_AddNumberToObject(resp, "node", node);
    cJSON_AddNumberToObject(resp, "index", index);
    cJSON_AddNumberToObject(resp, "sub", sub);

    // 将数据转为十六进制字符串
    char hex[16] = {0};
    for (size_t i = 0; i < data_len; i++) {
        sprintf(&hex[i * 2], "%02X", data[i]);
    }
    cJSON_AddStringToObject(resp, "data", hex);
    cJSON_AddNumberToObject(resp, "data_len", data_len);

    char *str = cJSON_PrintUnformatted(resp);
    strncpy(reply, str, reply_size - 1);
    free(str);
    cJSON_Delete(resp);
    return 0;
}

// 处理 SDO 写入
static int handle_sdo_write(cJSON *payload, char *reply, size_t reply_size)
{
    uint8_t node = cJSON_GetObjectItem(payload, "node")->valueint;
    uint16_t index = cJSON_GetObjectItem(payload, "index")->valueint;
    uint8_t sub = cJSON_GetObjectItem(payload, "sub")->valueint;
    int data_val = cJSON_GetObjectItem(payload, "data")->valueint;

    // 将 int 转为小端字节序
    uint8_t data[4];
    data[0] = data_val & 0xFF;
    data[1] = (data_val >> 8) & 0xFF;
    data[2] = (data_val >> 16) & 0xFF;
    data[3] = (data_val >> 24) & 0xFF;

    // 判断实际数据长度（1, 2, 或 4 字节）
    size_t len = 1;
    if (data_val > 0xFFFF) len = 4;
    else if (data_val > 0xFF) len = 2;

    int ret = canopen_sdo_write(node, index, sub, data, len, 1000);
    if (ret) {
        build_ack(reply, reply_size,
                  cJSON_GetObjectItem(payload, "seq")->valueint,
                  "error", "SDO 写入失败");
        return -1;
    }

    build_ack(reply, reply_size,
              cJSON_GetObjectItem(payload, "seq")->valueint,
              "ok", NULL);
    return 0;
}

// 处理点动
static int handle_jog(cJSON *payload, bool start, char *reply, size_t reply_size)
{
    uint8_t node = cJSON_GetObjectItem(payload, "node")->valueint;

    if (start) {
        cJSON *dir_item = cJSON_GetObjectItem(payload, "direction");
        cJSON *speed_item = cJSON_GetObjectItem(payload, "speed");

        const char *dir = dir_item ? dir_item->valuestring : "cw";
        int speed = speed_item ? speed_item->valueint : 500;

        // CiA 402 速度模式控制
        // 1. 设置运行模式为 Profile Velocity (3)
        uint8_t mode = 3;
        canopen_sdo_write(node, OD_INDEX_MODES_OF_OP, 0, &mode, 1, 500);

        // 2. 设置目标速度
        uint8_t speed_data[4];
        speed_data[0] = speed & 0xFF;
        speed_data[1] = (speed >> 8) & 0xFF;
        speed_data[2] = 0;
        speed_data[3] = 0;
        if (strcmp(dir, "ccw") == 0) {
            // 反转：负数用补码
            speed = -speed;
            speed_data[0] = speed & 0xFF;
            speed_data[1] = (speed >> 8) & 0xFF;
        }
        canopen_sdo_write(node, OD_INDEX_VELOCITY_TARGET, 0,
                          speed_data, 4, 500);

        // 3. 控制字使能：Shutdown(6) → Switch On(7) → Enable(0x0F)
        uint8_t cw[2] = {0x0F, 0x00};
        canopen_sdo_write(node, OD_INDEX_CONTROL_WORD, 0, cw, 2, 500);

        LOG_INF("点动启动: node=%d, dir=%s, speed=%d", node, dir, speed);
    } else {
        // 停止：设置控制字为 Quick Stop
        uint8_t cw[2] = {0x02, 0x00};
        canopen_sdo_write(node, OD_INDEX_CONTROL_WORD, 0, cw, 2, 500);
        LOG_INF("点动停止: node=%d", node);
    }

    return 0;
}

// 急停处理
static int handle_estop(char *reply, size_t reply_size)
{
    canopen_emergency_stop();
    build_ack(reply, reply_size, 0, "ok", "emergency_stop_executed");
    return 0;
}

int command_process(const char *json_str, size_t len,
                    char *reply_buf, size_t reply_size)
{
    cJSON *root = cJSON_ParseWithLength(json_str, len);
    if (!root) {
        LOG_ERR("JSON 解析失败");
        build_ack(reply_buf, reply_size, 0, "error", "invalid_json");
        return -1;
    }

    cJSON *cmd_item = cJSON_GetObjectItem(root, "cmd");
    if (!cmd_item) {
        cJSON_Delete(root);
        return -1;
    }

    const char *cmd = cmd_item->valuestring;
    int seq = 0;
    cJSON *seq_item = cJSON_GetObjectItem(root, "seq");
    if (seq_item) seq = seq_item->valueint;

    // ---- 心跳 ----
    if (strcmp(cmd, "heartbeat") == 0) {
        watchdog_feed();
        // 心跳不需要回复（减少网络流量）
        cJSON_Delete(root);
        return 0;
    }

    // ---- SDO 操作 ----
    cJSON *payload = cJSON_GetObjectItem(root, "payload");
    if (!payload) payload = root;  // 兼容：payload 可以省略

    int ret = 0;
    if (strcmp(cmd, "sdo_read") == 0) {
        ret = handle_sdo_read(payload, reply_buf, reply_size);
    } else if (strcmp(cmd, "sdo_write") == 0) {
        ret = handle_sdo_write(payload, reply_buf, reply_size);
    } else if (strcmp(cmd, "jog_start") == 0) {
        ret = handle_jog(payload, true, reply_buf, reply_size);
    } else if (strcmp(cmd, "jog_stop") == 0) {
        ret = handle_jog(payload, false, reply_buf, reply_size);
    } else if (strcmp(cmd, "estop") == 0) {
        ret = handle_estop(reply_buf, reply_size);
    } else if (strcmp(cmd, "ota_start") == 0) {
        ret = ota_handle_start(payload, reply_buf, reply_size);
    } else if (strcmp(cmd, "ota_chunk") == 0) {
        ret = ota_handle_chunk(payload, reply_buf, reply_size);
    } else if (strcmp(cmd, "ota_verify") == 0) {
        ret = ota_handle_verify(payload, reply_buf, reply_size);
    } else if (strcmp(cmd, "ota_flash") == 0) {
        ret = ota_handle_flash(payload, reply_buf, reply_size);
    } else {
        LOG_WRN("未知命令: %s", cmd);
        build_ack(reply_buf, reply_size, seq, "error", "unknown_command");
    }

    cJSON_Delete(root);
    return ret;
}
```

### 2.8.2 ota_manager.c

```c
#include "ota_manager.h"
#include "canopen_stack.h"
#include "udp_comm.h"
#include <zephyr/logging/log.h>
#include <zephyr/storage/flash_map.h>
#include <zephyr/sys/util.h>
#include <string.h>
#include <stdlib.h>
#include <mbedtls/md5.h>   // ESP32 自带 mbedTLS

LOG_MODULE_REGISTER(ota, CONFIG_LOG_DEFAULT_LEVEL);

static ota_context_t ota_ctx;

// OTA 刷写线程
static K_THREAD_STACK_DEFINE(ota_stack, 4096);
static struct k_thread ota_thread_data;

// 状态上报（发送给母机）
static void ota_report_state(const char *extra_msg)
{
    char buf[256];
    snprintf(buf, sizeof(buf),
             "{\"cmd\":\"ota_status\",\"state\":\"%s\","
             "\"progress\":%d,\"total\":%d,\"received\":%d}",
             ota_state_str(ota_ctx.state),
             ota_ctx.total_size > 0 ?
               (int)(ota_ctx.received_size * 100 / ota_ctx.total_size) : 0,
             ota_ctx.total_size,
             ota_ctx.received_size);
    udp_comm_send_json(buf);
}

static const char *ota_state_str(ota_state_t state)
{
    switch (state) {
    case OTA_STATE_IDLE:      return "idle";
    case OTA_STATE_RECEIVING: return "receiving";
    case OTA_STATE_VERIFYING: return "verifying";
    case OTA_STATE_FLASHING:  return "flashing";
    case OTA_STATE_DONE:      return "done";
    case OTA_STATE_ERROR:     return "error";
    default:                  return "unknown";
    }
}

int ota_manager_init(void)
{
    memset(&ota_ctx, 0, sizeof(ota_ctx));
    ota_ctx.state = OTA_STATE_IDLE;

    // 分配 Flash 缓冲区（假设固件最大 512KB）
    // 实际项目中应直接写入 Flash 分区
    ota_ctx.flash_buf = k_malloc(512 * 1024);
    if (!ota_ctx.flash_buf) {
        LOG_ERR("OTA 缓冲区分配失败");
        return -ENOMEM;
    }

    LOG_INF("OTA 管理器初始化完成");
    return 0;
}

// OTA 开始
int ota_handle_start(cJSON *payload, char *reply, size_t reply_size)
{
    if (ota_ctx.state != OTA_STATE_IDLE) {
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"OTA 已在进行中\"}");
        return -EBUSY;
    }

    cJSON *size_item = cJSON_GetObjectItem(payload, "size");
    cJSON *md5_item = cJSON_GetObjectItem(payload, "md5");

    if (!size_item || !md5_item) {
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"缺少 size 或 md5 参数\"}");
        return -EINVAL;
    }

    ota_ctx.total_size = size_item->valueint;
    ota_ctx.received_size = 0;
    ota_ctx.chunk_count = 0;
    strncpy(ota_ctx.expected_md5, md5_item->valuestring, 32);
    ota_ctx.expected_md5[32] = '\0';
    ota_ctx.state = OTA_STATE_RECEIVING;

    LOG_INF("OTA 开始: size=%d, md5=%s", ota_ctx.total_size, ota_ctx.expected_md5);

    snprintf(reply, reply_size,
             "{\"cmd\":\"ack\",\"status\":\"ok\","
             "\"msg\":\"OTA 传输开始\"}");
    ota_report_state(NULL);
    return 0;
}

// OTA 接收分片
int ota_handle_chunk(cJSON *payload, char *reply, size_t reply_size)
{
    if (ota_ctx.state != OTA_STATE_RECEIVING) {
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"不在接收状态\"}");
        return -EINVAL;
    }

    cJSON *offset_item = cJSON_GetObjectItem(payload, "offset");
    cJSON *data_item = cJSON_GetObjectItem(payload, "data");

    if (!offset_item || !data_item) {
        return -EINVAL;
    }

    uint32_t offset = offset_item->valueint;
    const char *b64_data = data_item->valuestring;

    // Base64 解码（ESP32 mbedTLS 支持）
    size_t decoded_len = 0;
    mbedtls_base64_decode(
        ota_ctx.flash_buf + offset,
        ota_ctx.total_size - offset,
        &decoded_len,
        (const uint8_t *)b64_data,
        strlen(b64_data)
    );

    ota_ctx.received_size += decoded_len;
    ota_ctx.chunk_count++;

    LOG_INF("OTA 分片: offset=%d, decoded=%d, total=%d/%d",
            offset, decoded_len, ota_ctx.received_size, ota_ctx.total_size);

    // 每 10 个分片上报一次进度
    if (ota_ctx.chunk_count % 10 == 0) {
        ota_report_state(NULL);
    }

    snprintf(reply, reply_size,
             "{\"cmd\":\"ack\",\"status\":\"ok\",\"offset\":%d}", offset);
    return 0;
}

// OTA MD5 校验
int ota_handle_verify(cJSON *payload, char *reply, size_t reply_size)
{
    if (ota_ctx.state != OTA_STATE_RECEIVING) {
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"不在接收状态\"}");
        return -EINVAL;
    }

    if (ota_ctx.received_size != ota_ctx.total_size) {
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"数据不完整: %d/%d\"}",
                 ota_ctx.received_size, ota_ctx.total_size);
        return -EINVAL;
    }

    ota_ctx.state = OTA_STATE_VERIFYING;
    ota_report_state(NULL);

    // 计算 MD5
    mbedtls_md5_context md5_ctx;
    mbedtls_md5_init(&md5_ctx);
    mbedtls_md5_starts(&md5_ctx);
    mbedtls_md5_update(&md5_ctx, ota_ctx.flash_buf, ota_ctx.received_size);

    unsigned char digest[16];
    mbedtls_md5_finish(&md5_ctx, digest);
    mbedtls_md5_free(&md5_ctx);

    // 转为十六进制字符串
    char md5_str[33] = {0};
    for (int i = 0; i < 16; i++) {
        sprintf(&md5_str[i * 2], "%02x", digest[i]);
    }

    LOG_INF("OTA MD5 校验: 计算=%s, 预期=%s", md5_str, ota_ctx.expected_md5);

    if (strcmp(md5_str, ota_ctx.expected_md5) != 0) {
        LOG_ERR("MD5 不匹配!");
        ota_ctx.state = OTA_STATE_ERROR;
        snprintf(reply, reply_size,
                 "{\"cmd\":\"ack\",\"status\":\"error\","
                 "\"msg\":\"MD5 校验失败\"}");
        ota_report_state("md5_mismatch");
        return -EIO;
    }

    LOG_INF("MD5 校验通过!");
    ota_ctx.state = OTA_STATE_IDLE;  // 等待 flash 命令
    snprintf(reply, reply_size,
             "{\"cmd\":\"ack\",\"status\":\"ok\","
             "\"msg\":\"MD5 校验通过\"}");
    return 0;
}

// OTA 刷写电机（在线程中执行，避免阻塞主线程）
static void ota_flash_thread(void *p1, void *p2, void *p3)
{
    ARG_UNUSED(p1); ARG_UNUSED(p2); ARG_UNUSED(p3);

    uint8_t target_node = 1;  // 默认目标节点

    LOG_INF("开始刷写电机固件，目标节点=%d", target_node);

    // 1. 使目标节点进入 Boot 模式（通过 NMT 或 SDO）
    uint8_t mode = 0x00;  // bootloader 模式（具体值取决于电机控制器）
    canopen_sdo_write(target_node, 0x1F50, 1, &mode, 1, 2000);

    // 2. SDO Block Download 分段写入
    // 这里简化为按 4 字节块传输，实际应使用 SDO Block Transfer
    uint32_t offset = 0;
    uint32_t chunk_size = 4;  // SDO 快速下载最多 4 字节
    int ret = 0;

    while (offset < ota_ctx.received_size) {
        size_t len = MIN(chunk_size, ota_ctx.received_size - offset);
        ret = canopen_sdo_write(target_node, 0x1F50, 2,
                                ota_ctx.flash_buf + offset, len, 2000);
        if (ret) {
            LOG_ERR("刷写失败 at offset=%d", offset);
            ota_ctx.state = OTA_STATE_ERROR;
            ota_report_state("flash_error");
            return;
        }
        offset += len;

        // 每 1KB 上报进度
        if (offset % 1024 == 0) {
            LOG_INF("刷写进度: %d/%d", offset, ota_ctx.received_size);
        }
    }

    // 3. 触发电机重启应用新固件
    canopen_nmt_send(CANOPEN_NMT_RESET, target_node);

    LOG_INF("电机固件刷写完成!");
    ota_ctx.state = OTA_STATE_DONE;
    ota_report_state(NULL);
}

int ota_handle_flash(cJSON *payload, char *reply, size_t reply_size)
{
    ota_ctx.state = OTA_STATE_FLASHING;
    ota_report_state(NULL);

    // 在独立线程中执行刷写
    k_thread_create(&ota_thread_data, ota_stack,
                    K_THREAD_STACK_SIZEOF(ota_stack),
                    ota_flash_thread, NULL, NULL, NULL,
                    K_PRIO_PREEMPT(10), 0, K_NO_WAIT);
    k_thread_name_set(&ota_thread_data, "ota_flash");

    snprintf(reply, reply_size,
             "{\"cmd\":\"ack\",\"status\":\"ok\","
             "\"msg\":\"开始刷写\"}");
    return 0;
}

ota_state_t ota_get_state(void)
{
    return ota_ctx.state;
}
```

### 3.3.1 network/protocol.py — 消息协议封装

```python
"""protocol.py — 母机与子机之间的 JSON 协议封装"""

import json
import time
from typing import Optional, Dict, Any


class Protocol:
    """封装所有通信协议的消息格式"""

    _seq_counter = 0

    @classmethod
    def _next_seq(cls) -> int:
        cls._seq_counter += 1
        return cls._seq_counter

    @staticmethod
    def _build(cmd: str, payload: Optional[Dict] = None) -> str:
        """构建标准消息"""
        msg = {
            "cmd": cmd,
            "seq": Protocol._next_seq(),
            "ts": int(time.time()),
        }
        if payload:
            msg["payload"] = payload
        return json.dumps(msg, separators=(',', ':'))

    # ---- 心跳 ----
    @staticmethod
    def heartbeat() -> str:
        return Protocol._build("heartbeat")

    # ---- SDO 操作 ----
    @staticmethod
    def sdo_read(node: int, index: int, sub: int = 0) -> str:
        return Protocol._build("sdo_read", {
            "node": node, "index": index, "sub": sub
        })

    @staticmethod
    def sdo_write(node: int, index: int, sub: int, data: int) -> str:
        return Protocol._build("sdo_write", {
            "node": node, "index": index, "sub": sub, "data": data
        })

    # ---- 电机控制 ----
    @staticmethod
    def enable(node: int) -> str:
        return Protocol._build("enable", {"node": node})

    @staticmethod
    def disable(node: int) -> str:
        return Protocol._build("disable", {"node": node})

    @staticmethod
    def estop() -> str:
        return Protocol._build("estop")

    @staticmethod
    def jog_start(node: int, direction: str = "cw", speed: int = 500) -> str:
        return Protocol._build("jog_start", {
            "node": node, "direction": direction, "speed": speed
        })

    @staticmethod
    def jog_stop(node: int) -> str:
        return Protocol._build("jog_stop", {"node": node})

    # ---- OTA ----
    @staticmethod
    def ota_start(size: int, md5: str) -> str:
        return Protocol._build("ota_start", {"size": size, "md5": md5})

    @staticmethod
    def ota_chunk(offset: int, data_b64: str) -> str:
        return Protocol._build("ota_chunk", {
            "offset": offset, "data": data_b64
        })

    @staticmethod
    def ota_verify() -> str:
        return Protocol._build("ota_verify")

    @staticmethod
    def ota_flash(node: int = 1) -> str:
        return Protocol._build("ota_flash", {"node": node})

    # ---- 解析回复 ----
    @staticmethod
    def parse(data: str) -> Dict[str, Any]:
        try:
            return json.loads(data)
        except json.JSONDecodeError:
            return {"cmd": "unknown", "error": "json_parse_failed"}
```

### 3.3.2 network/udp_client.py — UDP 通信客户端

```python
"""udp_client.py — UDP 通信客户端，支持异步收发"""

import socket
import threading
import logging
from typing import Optional
from PyQt5.QtCore import QObject, pyqtSignal

from config import DONGLE_IP, DONGLE_UDP_PORT, LOCAL_UDP_PORT


logger = logging.getLogger(__name__)


class UDPClient(QObject):
    """与 CAN Dongle 的 UDP 通信客户端

    使用 Qt 信号将收到的数据传递给 UI 线程，避免线程安全问题。
    """

    data_received = pyqtSignal(str)       # 收到 Dongle 数据
    connection_changed = pyqtSignal(bool)  # 连接状态变化

    def __init__(self, parent=None):
        super().__init__(parent)
        self._sock: Optional[socket.socket] = None
        self._running = False
        self._recv_thread: Optional[threading.Thread] = None
        self._connected = False

    def start(self) -> bool:
        try:
            self._sock = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
            self._sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
            self._sock.settimeout(0.5)
            self._sock.bind(('0.0.0.0', LOCAL_UDP_PORT))

            self._running = True
            self._recv_thread = threading.Thread(
                target=self._recv_loop, daemon=True
            )
            self._recv_thread.start()
            logger.info(f"UDP 客户端启动，监听端口 {LOCAL_UDP_PORT}")
            return True
        except Exception as e:
            logger.error(f"UDP 客户端启动失败: {e}")
            return False

    def stop(self):
        self._running = False
        if self._sock:
            self._sock.close()
            self._sock = None

    def send(self, data: str):
        if not self._sock:
            return False
        try:
            self._sock.sendto(
                data.encode('utf-8'),
                (DONGLE_IP, DONGLE_UDP_PORT)
            )
            return True
        except Exception as e:
            logger.error(f"发送失败: {e}")
            return False

    def _recv_loop(self):
        while self._running:
            try:
                data, addr = self._sock.recvfrom(2048)
                if data:
                    msg = data.decode('utf-8')
                    self.data_received.emit(msg)
                    if not self._connected:
                        self._connected = True
                        self.connection_changed.emit(True)
            except socket.timeout:
                continue
            except OSError:
                if self._running:
                    logger.error("Socket 异常关闭")
                break
        self._connected = False
        self.connection_changed.emit(False)
```

### core/motor_data.py

```python
"""motor_data.py — 电机实时数据存储与更新"""

import time
from collections import deque
from typing import Dict, List, Tuple
from PyQt5.QtCore import QObject, pyqtSignal

from config import WAVEFORM_HISTORY


class MotorData(QObject):
    """电机数据模型，支持实时波形记录"""

    data_updated = pyqtSignal()

    def __init__(self, parent=None):
        super().__init__(parent)

        # 当前值
        self.current: float = 0.0
        self.voltage: float = 0.0
        self.speed: int = 0
        self.position: float = 0.0
        self.torque: float = 0.0
        self.status_word: int = 0
        self.fault_code: int = 0
        self.mode: int = 0
        self.alive: bool = False
        self.wdg_ms: int = 0

        # 历史数据（波形）
        self._history_size = WAVEFORM_HISTORY
        self._timestamps: deque = deque(maxlen=self._history_size)
        self._current_history: deque = deque(maxlen=self._history_size)
        self._speed_history: deque = deque(maxlen=self._history_size)
        self._torque_history: deque = deque(maxlen=self._history_size)
        self._start_time = time.time()

    def update_from_dict(self, data: Dict):
        """从子机上报的 JSON 数据更新"""
        if "current" in data: self.current = data["current"]
        if "voltage" in data: self.voltage = data["voltage"]
        if "speed" in data: self.speed = data["speed"]
        if "position" in data: self.position = data["position"]
        if "torque" in data: self.torque = data["torque"]
        if "fault" in data: self.fault_code = data["fault"]
        if "status_word" in data:
            try: self.status_word = int(data["status_word"], 16)
            except (ValueError, TypeError): pass
        if "alive" in data: self.alive = data["alive"]
        if "wdg_ms" in data: self.wdg_ms = data["wdg_ms"]

        t = time.time() - self._start_time
        self._timestamps.append(t)
        self._current_history.append(self.current)
        self._speed_history.append(self.speed)
        self._torque_history.append(self.torque)
        self.data_updated.emit()

    def get_waveform_data(self, param: str) -> Tuple[List[float], List[float]]:
        x = list(self._timestamps)
        if param == "current": return x, list(self._current_history)
        elif param == "speed": return x, list(self._speed_history)
        elif param == "torque": return x, list(self._torque_history)
        return x, []

    def get_status_text(self) -> str:
        sw = self.status_word
        if sw & 0x004F == 0x0000: return "未准备好"
        if sw & 0x004F == 0x0040: return "已关闭"
        if sw & 0x006F == 0x0021: return "已准备好"
        if sw & 0x006F == 0x0023: return "已开机"
        if sw & 0x006F == 0x0027: return "已使能"
        if sw & 0x004F == 0x0008: return "故障"
        return f"未知(0x{sw:04X})"

    def is_fault(self) -> bool:
        return (self.status_word & 0x0008) != 0 or self.fault_code != 0
```

```python
            self, "选择电机固件文件", "",
            "固件文件 (*.bin *.hex *.fw *.motor);;所有文件 (*)"
        )
        if not path: return
        with open(path, 'rb') as f:
            self._firmware_data = f.read()
        self._firmware_md5 = hashlib.md5(self._firmware_data).hexdigest()
        size_kb = len(self._firmware_data) / 1024
        self.file_label.setText(os.path.basename(path))

    def _start_transfer(self):
        if not self._firmware_data: return
        self._sending = True
        self.send_btn.setEnabled(False)
        self.progress_bar.setValue(0)
        self._log("开始传输固件到子机...")
        threading.Thread(target=self._send_firmware_thread, daemon=True).start()

    def _send_firmware_thread(self):
        total_size = len(self._firmware_data)
        # Step 1: OTA 开始
        self.udp_client.send(Protocol.ota_start(total_size, self._firmware_md5))
        time.sleep(0.1)
        # Step 2: 分片发送
        offset = 0
        chunk_count = 0
        start_time = time.time()
        while offset < total_size and self._sending:
            chunk = self._firmware_data[offset:offset + OTA_CHUNK_SIZE]
            chunk_b64 = base64.b64encode(chunk).decode('ascii')
            self.udp_client.send(Protocol.ota_chunk(offset, chunk_b64))
            offset += len(chunk)
            chunk_count += 1
            progress = int(offset * 100 / total_size)
            elapsed = time.time() - start_time
            speed = (offset / 1024) / elapsed if elapsed > 0 else 0
            remaining = (total_size - offset) / (speed * 1024) if speed > 0 else 0
            QTimer.singleShot(0, lambda p=progress, s=speed, r=remaining:
                self._update_progress(p, s, r))
            time.sleep(OTA_SEND_INTERVAL)
        if self._sending:
            elapsed = time.time() - start_time
            self._log(f"传输完成! 耗时 {elapsed:.1f}s, 共 {chunk_count} 个分片")
            QTimer.singleShot(0, self._on_transfer_complete)

    def _update_progress(self, progress, speed, remaining):
        self.progress_bar.setValue(progress)
        self.progress_bar.setFormat(f"{progress}%")
        self.speed_label.setText(f"速度: {speed:.1f} KB/s")
        self.eta_label.setText(f"剩余: {remaining:.0f}s")

    def _on_transfer_complete(self):
        self.send_btn.setEnabled(True)
        self.verify_btn.setEnabled(True)
        self.progress_bar.setFormat("传输完成，等待校验")

    def _verify_firmware(self):
        self._log("请求子机进行 MD5 校验...")
        self.udp_client.send(Protocol.ota_verify())

    def _flash_motor(self):
        reply = QMessageBox.warning(
            self, "确认刷写",
            "即将开始刷写电机固件，请确保：\n\n"
            "1. 电机处于安全状态\n2. CAN 总线连接稳定\n3. 不会断电\n\n是否继续？",
            QMessageBox.Yes | QMessageBox.No
        )
        if reply != QMessageBox.Yes: return
        self._log("开始刷写电机固件...")
        self.flash_btn.setEnabled(False)
        self.udp_client.send(Protocol.ota_flash(1))

    def handle_reply(self, data: dict):
        cmd = data.get("cmd", "")
        if cmd == "ota_status":
            state = data.get("state", "")
            if state == "verifying":
                self._log("子机正在校验固件...")
                self.progress_bar.setFormat("校验中...")
            elif state == "done":
                self._log("电机固件刷写完成!")
                self.flash_status.setText("状态: 刷写完成 ✓")
                self.flash_btn.setEnabled(True)
            elif state == "error":
                self._log(f"OTA 错误")
                self.flash_status.setText("状态: 错误 ✗")
                self.flash_btn.setEnabled(True)
            elif state == "idle" and self.verify_btn.isEnabled():
                self._log("MD5 校验通过! 可以开始刷写")
                self.flash_btn.setEnabled(True)
                self.flash_status.setText("状态: 校验通过，可以刷写")
        elif cmd == "ack":
            status = data.get("status", "")
            msg = data.get("msg", "")
            self._log(f"{'✓' if status == 'ok' else '✗'} {msg}")
```

```python
"""button_handler.py — GPIO 物理按键（带硬件消抖 + 最高优先级）"""

import time
from PyQt5.QtCore import QObject, pyqtSignal

from config import (
    BTN_ENABLE_GPIO, BTN_JOG_CW_GPIO, BTN_JOG_CCW_GPIO,
    BTN_CONFIRM_GPIO, BTN_DEBOUNCE_MS
)


class ButtonHandler(QObject):
    button_pressed = pyqtSignal(str)

    def __init__(self, parent=None):
        super().__init__(parent)
        self._last_press_time = {}
        self._has_gpio = False
        self._setup_gpio()

    def _setup_gpio(self):
        try:
            import RPi.GPIO as GPIO
            self._gpio = GPIO
            self._has_gpio = True
            GPIO.setmode(GPIO.BCM)
            GPIO.setwarnings(False)

            buttons = {
                BTN_ENABLE_GPIO: "enable",
                BTN_JOG_CW_GPIO: "jog_cw",
                BTN_JOG_CCW_GPIO: "jog_ccw",
                BTN_CONFIRM_GPIO: "confirm",
            }
            for gpio, name in buttons.items():
                GPIO.setup(gpio, GPIO.IN, pull_up_down=GPIO.PUD_UP)
                GPIO.add_event_detect(
                    gpio, GPIO.FALLING,
                    callback=lambda ch, n=name: self._on_interrupt(n),
                    bouncetime=BTN_DEBOUNCE_MS
                )
                self._last_press_time[name] = 0
            print("GPIO 按键初始化成功")
        except (ImportError, RuntimeError):
            self._has_gpio = False
            print("未检测到 GPIO，物理按键已禁用")

    def _on_interrupt(self, button_name: str):
        now = time.monotonic() * 1000
        last = self._last_press_time.get(button_name, 0)
        if (now - last) < BTN_DEBOUNCE_MS:
            return
        self._last_press_time[button_name] = now
        self.button_pressed.emit(button_name)
```

## 3.6 Wi-Fi 自动连接脚本

```bash
#!/bin/bash
# /home/pi/connect_dongle.sh — 开机自动连接 CAN Dongle 热点

SSID="CAN_Dongle_01"
PASSWORD="C@nDongle2024"
MAX_RETRIES=30

echo "正在扫描 CAN Dongle 热点..."
for i in $(seq 1 $MAX_RETRIES); do
    nmcli device wifi rescan 2>/dev/null
    sleep 1
    nmcli device wifi connect "$SSID" password "$PASSWORD" 2>/dev/null
    if [ $? -eq 0 ]; then
        echo "已连接到 $SSID"
        sleep 2
        ping -c 1 -W 2 192.168.4.1 > /dev/null 2>&1
        if [ $? -eq 0 ]; then
            echo "CAN Dongle 通信正常"
            exit 0
        fi
    fi
    echo "重试 $i/$MAX_RETRIES..."
    sleep 2
done
echo "无法连接到 CAN Dongle"
exit 1
```

设置开机自启：

```bash
chmod +x /home/pi/connect_dongle.sh
(crontab -l; echo "@reboot /home/pi/connect_dongle.sh") | crontab -
```

```
                120Ω
               ┌──┤├──┐

  CAN_H ───────┤       ├─── CAN_H
```

## 4.5 调试技巧与常见问题

### 4.5.1 常见问题排查

**问题 1：ESP32 Wi-Fi AP 启动失败**
```
[ERR] Wi-Fi AP 启动失败: -1
```
- 检查 ESP32 天线是否接好
- 确认固件配置中 `CONFIG_WIFI=y` 已启用
- 检查是否有其他 Wi-Fi 干扰

**问题 2：母机连不上 Dongle 热点**
```bash
# 检查 Wi-Fi 扫描结果
nmcli device wifi list | grep CAN_Dongle

# 手动连接
nmcli device wifi connect CAN_Dongle_01 password C@nDongle2024

# 如果连接失败，检查密码和安全类型
nmcli connection show
```

**问题 3：CAN 总线无通信**
```bash
# 用 can-utils 测试（如果 ESP32 暴露了 CAN 接口）
sudo apt install -y can-utils
sudo ip link set can0 type can bitrate 1000000
sudo ip link set up can0
candump can0  # 监听所有 CAN 帧
```

- 检查 CAN_H 和 CAN_L 是否接反
- 确认 120Ω 终端电阻已安装
- 确认波特率与电机驱动器一致（通常 500kbps 或 1Mbps）

**问题 4：SDO 通信超时**
```
[canopen] SDO 读超时 (节点 1, 索引 0x1000)
```
- 确认电机驱动器已上电
- 确认节点 ID 匹配
- 检查 CAN 波特率是否一致
- 用 `candump` 看是否有 CAN 错误帧

**问题 5：波形不更新**
- 检查母机是否收到 `motor_status` 消息
- 在终端运行 `python main.py 2>&1 | grep motor_status` 查看
- 确认子机在定期发送状态上报


**母机（Python）：** 修改 `main.py`

### 6.2.5 物理按键适配

掌机的物理按键通常映射为 `/dev/input/eventX` 设备，需要修改 `button_handler.py`：

```python
# 在 button_handler.py 中添加 evdev 支持（替代 RPi.GPIO）
# 安装：pip3 install evdev

import evdev

def _setup_evdev(self):
    """掌机按键通过 evdev 读取"""
    devices = [evdev.InputDevice(path) for path in evdev.list_devices()]
    for device in devices:
        if 'gpio' in device.name.lower() or 'gamepad' in device.name.lower():
            print(f"找到输入设备: {device.name}")
            for event in device.read_loop():
                if event.type == evdev.ecodes.EV_KEY:
                    key_event = evdev.categorize(event)
                    if key_event.keystate == 1:  # 按下
                        self._map_and_emit(key_event.scancode)
```

> **注意：** 不同掌机的按键映射不同，需要根据具体型号调整 scancode 到功能的映射关系。可以通过 `evtest` 工具查看按键事件。

## A. CANopen CiA 402 状态机快速参考

```
       ┌─────────────┐
       │ Not Ready    │
       │ to Switch On │
       └──────┬───────┘
              │ (内部自动转换)
       ┌──────▼───────┐
       │ Switch On     │
       │ Disabled      │ ← 控制字: Shutdown (0x0006)
       └──────┬───────┘
              │
       ┌──────▼───────┐
       │ Ready to      │ ← 状态字: 0x0021
       │ Switch On     │ ← 控制字: Switch On (0x0007)
       └──────┬───────┘
              │
       ┌──────▼───────┐
       │ Switched On   │ ← 状态字: 0x0023
       │               │ ← 控制字: Enable (0x000F)
       └──────┬───────┘
              │
       ┌──────▼───────┐
       │ Operation     │ ← 状态字: 0x0027 ✓
       │ Enabled       │    (电机可运行)
       └──────┬───────┘
              │
       ┌──────▼───────┐
       │ Quick Stop    │ ← 控制字: Quick Stop (0x0002)
       │ Active        │
       └──────────────┘

任何状态 → Fault: 状态字 bit3 = 1
Fault → Fault Reset: 控制字 Fault Reset (0x0080)
```
