# Dongle 部分 0 基础操作教程

这份教程默认你完全没有嵌入式、Zephyr、VSCode、终端经验。

先记住一句话：

```text
VSCode 用来新建/查看/修改文件。
终端用来输入命令，比如编译、烧录、查看设备。
```

## 一、这个文件夹到底在哪里

你的项目根目录是：

```text
/Users/guoweifeng/GameBoy
```

dongle 部分应该放在：

```text
/Users/guoweifeng/GameBoy/dongle_firmware
```

后面真正的 ESP32 固件工程要放在：

```text
/Users/guoweifeng/GameBoy/dongle_firmware/can_dongle
```

## 二、`can_dongle` 文件夹是在 VSCode 里创建吗？

可以。

对你现在的学习阶段，我建议：

```text
用 VSCode 创建和查看文件。
用终端执行编译和烧录命令。
```

原因是：

- VSCode 里能直观看到文件夹结构。
- 你不容易把文件建错地方。
- 以后写 C 代码、看报错、搜索文件都更方便。

不过实际开发时，我也可以直接帮你创建这个文件夹和里面的代码。你不需要自己手动建每一个 `.c`、`.h` 文件。

## 三、如何打开这个项目文件夹

### 方法 A：从 VSCode 打开

1. 打开 VSCode。
2. 看屏幕最上方菜单栏。
3. 点击 `File`。
4. 点击 `Open Folder...`。
5. 在弹出的窗口里找到你的用户目录。
6. 依次进入：

```text
Users
guoweifeng
GameBoy
```

7. 选中 `GameBoy` 文件夹。
8. 点击 `Open`。
9. 如果 VSCode 弹出提示，问是否信任这个文件夹，点击 `Trust` 或 `Yes`。

完成后，VSCode 左侧应该能看到这些文件夹：

```text
godot_terminal
handheld_terminal
mock_server
memory
dongle_firmware
```

如果你看到了这些，说明打开的是正确项目。

### 方法 B：从终端打开 VSCode

如果你已经会打开终端，可以输入：

```sh
code "/Users/guoweifeng/GameBoy"
```

如果提示 `code: command not found`，说明 VSCode 的命令行工具还没安装。先不用管，用方法 A 就行。

## 四、如何在 VSCode 里创建 `can_dongle` 文件夹

如果我还没有帮你创建，那么你可以这样做：

1. 打开 VSCode。
2. 左侧找到 `dongle_firmware` 文件夹。
3. 鼠标移动到 `dongle_firmware` 上。
4. 右键点击 `dongle_firmware`。
5. 点击 `New Folder`。
6. 输入：

```text
can_dongle
```

7. 按键盘 `Enter`。

完成后，左侧应该看到：

```text

```

## 五、如何在 VSCode 里新建文件

假设你要在 `can_dongle` 下面新建 `CMakeLists.txt`：

1. 鼠标右键点击 `can_dongle` 文件夹。
2. 点击 `New File`。
3. 输入：

```text
CMakeLists.txt
```

4. 按键盘 `Enter`。
5. 中间编辑区会打开这个文件。
6. 你可以在里面输入内容。
7. 保存文件：
   - Mac 快捷键：`Command + S`
   - 或者菜单栏点击 `File` -> `Save`

如果要新建 `src/main.c`，需要先建 `src` 文件夹：

1. 右键点击 `can_dongle`。
2. 点击 `New Folder`。
3. 输入：

```text
src
```

4. 按 `Enter`。
5. 右键点击 `src`。
6. 点击 `New File`。
7. 输入：

```text
main.c
```

8. 按 `Enter`。

## 六、我建议你不要手动创建所有源码文件

dongle 固件会有很多文件：

```text
CMakeLists.txt
prj.conf
boards/esp32s3_devkitm.overlay
src/main.c
src/udp_comm.c
src/udp_comm.h
src/can_raw.c
src/can_raw.h
src/json_protocol.c
src/json_protocol.h
src/watchdog.c
src/watchdog.h
```

你可以手动建，但很容易：

- 文件名打错。
- 文件夹层级建错。
- `.h` 和 `.c` 放错位置。
- 忘记保存。

所以实际开发时，建议让我直接帮你生成这个工程骨架。你只需要学会打开、查看、保存、编译和烧录。

在终端输入：


然后按 `Enter`。

## 十一、怎么用终端创建 `can_dongle` 文件夹

如果不用 VSCode，也可以用终端创建。

先进入项目目录：

```sh
cd "/Users/guoweifeng/GameBoy"
```

再输入：

```sh
mkdir -p dongle_firmware/can_dongle
```

然后按 `Enter`。

这条命令意思是：

- `mkdir`：创建文件夹。
- `-p`：如果中间文件夹不存在，也一起创建；如果已经存在，不报错。
- `dongle_firmware/can_dongle`：要创建的路径。

然后输入：

```sh
ls dongle_firmware
```

如果看到：

```text
can_dongle
README.md
零基础操作教程.md
```

说明创建成功。

## 十二、编译 dongle 固件在哪里输入

编译命令不是在 VSCode 文件里输入，而是在 Mac 终端里输入。

以后 `can_dongle` 源码写好以后，打开终端，输入：

```sh
cd /Users/guoweifeng/esp32-can-dongle
```

按 `Enter`。

然后输入：

```sh
source /Users/guoweifeng/zephyrproject/.venv/bin/activate
```

按 `Enter`。

这一步是进入 Zephyr 的 Python 环境。

然后输入：

```sh
west build -b esp32s3_devkitm "/Users/guoweifeng/GameBoy/dongle_firmware/can_dongle" -d build-can-dongle
```

按 `Enter`。

如果成功，最后会看到类似：

```text
Memory region
FLASH
RAM
```

或者看到 build finished / succeeded 之类的信息。

## 十三、烧录 dongle 固件在哪里输入

烧录也在 Mac 终端输入。

先把 ESP32-S3 通过 USB 线插到 Mac。

然后终端输入：

```sh
cd /Users/guoweifeng/esp32-can-dongle
```

按 `Enter`。

再输入：

```sh
source /Users/guoweifeng/zephyrproject/.venv/bin/activate
```

按 `Enter`。

最后输入：

```sh
west flash -d build-can-dongle
```

按 `Enter`。

如果烧录成功，ESP32-S3 会重启，运行新的 dongle 固件。

## 十四、第一次开发时你需要理解的几个词

### RGB30

你的掌机。它运行 Godot UI，是“母机”。

### Dongle

ESP32-S3 CAN Dongle。它连接 Wi-Fi 和 CAN 总线，是“子机”。

### UDP

RGB30 和 dongle 之间的网络通信方式。它速度快，适合 150 ms 心跳和实时状态。

### JSON

RGB30 和 dongle 之间传输的数据格式，例如：

```json
{
  "cmd": "enable",
  "payload": {
    "node": 1
  }
}
```

### CAN

电机总线。真实电机数据会从这里来。

### CANopen

建立在 CAN 上的电机控制协议。对象字典、SDO、PDO、NMT、CiA 402 都属于这个世界。

### Zephyr

运行在 ESP32-S3 上的实时操作系统。dongle 固件会基于 Zephyr 写。

### west

Zephyr 的构建工具。你可以理解成“用来编译和烧录 Zephyr 项目的命令”。
